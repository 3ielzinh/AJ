Priorizar layouts amplos, leitura clara e uso equilibrado do espaço lateral.

Evitar interfaces genéricas ou excessivamente centralizadas quando a tela comportar melhor distribuição horizontal.

Antes de criar CSS novo por página, verificar se o comportamento pode ser resolvido com classes e padrões já existentes.

Manter consistência entre Home, Perfil, Monitoramento, Dashboard, Reordenar e formulários.

Usar hierarquia visual clara: hero, bloco de contexto, filtros, conteúdo principal e ações.

Evitar estilos inline, salvo casos realmente pontuais e justificados.

Em páginas com filtros, alinhar navegação, ações e busca como um único sistema visual.

Após alterações de interface, validar o HTML/template para evitar duplicação de blocos, tags desalinhadas ou CSS misturado com markup.
