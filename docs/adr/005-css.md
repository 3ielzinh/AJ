# ADR 005: CSS Modular Sem Bootstrap

## Status

Aceita

## Contexto

O projeto ja possui CSS proprio organizado por tokens, base e componentes. Introduzir Bootstrap ou CSS inline criaria concorrencia de padroes e aumentaria o custo de manutencao visual.

## Decisao

Usar CSS modular proprio e evitar Bootstrap. Estilos novos devem ser adicionados preferencialmente em:

- `static/css/components.css`, quando forem reutilizaveis;
- `static/css/base.css`, quando forem fundacionais;
- arquivos especificos apenas quando houver necessidade clara e escopo bem delimitado.

## Consequencias

- O visual permanece coerente e controlado.
- A evolucao do design system fica mais simples.
- Novos componentes exigem disciplina para serem genericos o suficiente, sem acoplar regra de uma pagina especifica.
